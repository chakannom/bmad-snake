# Snake Map Balance Tuning v1

## 목표
- 맵 1~20의 난이도 곡선을 급격한 스파이크 없이 점진적으로 유지
- 클리어율 기준으로 시간/목표점수 조정 규칙을 표준화

## 기준 지표
- `clear_rate`: 맵 시도 대비 클리어 비율
- `avg_survival_sec`: 평균 생존 시간
- `avg_score`: 평균 점수
- `p95_input_ms`: 입력 지연 p95

## 조정 규칙
- 기준 샘플: 맵별 최소 30회 시도
- 너무 어려움: `clear_rate < 35%`
  - `targetScore -1` 또는 `timeLimit +5s` (한 번에 한 항목만)
- 너무 쉬움: `clear_rate > 75%`
  - `targetScore +1` 또는 `timeLimit -5s` (한 번에 한 항목만)
- 유지 구간: `35% <= clear_rate <= 75%`
  - 조정 없음

## 난이도 스파이크 방지 규칙
- 인접 맵 기준 `clear_rate` 차이 절대값 20%p 초과 시 조정 검토
- 인접 맵 간 `targetScore` 증가폭은 기본 1~2 유지
- 동일 단계(1~5, 6~10, 11~15, 16~20) 내 급격한 장애물 증가 금지

## 품질 게이트
- p95 입력 지연 100ms 초과 시 난이도 조정보다 입력/성능 이슈 우선 해결
- 평균 FPS 55 미만 구간에서는 맵 난이도 데이터 확정 보류

## 운영 루프
1. 주간 플레이 로그 수집
2. 맵별 지표 집계
3. 규칙 기반 1차 조정안 산출
4. 내부 플레이테스트 10세션 검증
5. 다음 빌드 반영
