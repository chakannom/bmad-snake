# Snake Map Balance Tuning v1

## 목적
- 이 문서는 현재 코드(`src/maps/mapDefinitions.ts`, `src/game/Scoring.ts`)에 구현된 밸런스 기준을 기록한다.
- 임의 목표치가 아니라, 실제 런타임 계산식과 규칙을 기준으로 튜닝한다.

## 전역 밸런스 상수
- 보드 크기: `20 x 20`
- 총 스테이지 수: `20`
- 먹이 1개 점수: `+10` (`stageScore`, `totalScore` 동시 증가)
- 최소 틱: `MIN_TICK_MS = 52`
- 먹이 1개당 가속: `SPEED_UP_PER_FOOD_MS = 2`
- 사망 페널티: `DEATH_PENALTY_RATIO = 0.3` (현재 점수의 30% 차감, `Math.floor`)

## 스테이지 생성 공식
- `level = 1..20`
- `targetScore = 40 + level * 10`
- `timeLimitSec = Math.max(18, 34 - Math.floor(level / 2))`
- `tickMs = Math.max(70, 125 - level * 2)`

## 게임 진행 시 밸런스 규칙
- 타이머 감소는 매 스텝마다 `state.remainingMs -= stage.tickMs`로 계산한다.
- 속도 가속은 `state.currentTickMs`에만 반영된다.
- 따라서 먹이로 가속되어도 제한 시간 감소량은 기본 `stage.tickMs` 기준이다.
- 먹이 획득 시 `state.stageScore >= stage.targetScore`면 즉시 스테이지 클리어.

## 사망/재시작 규칙
- 벽/자기몸/장애물 충돌 시 사망 처리.
- 사망 시 즉시 점수 페널티 적용:
  - `stageScore = floor(stageScore * 0.7)`
  - `totalScore = floor(totalScore * 0.7)`
- 같은 스테이지를 즉시 재시작하며:
  - 스네이크/방향/남은 시간 초기화
  - 속도는 해당 맵 기본 `tickMs`로 리셋
  - `stageScore`는 페널티 적용값을 유지

## 장애물 스케일링 규칙
- 레벨별 고정 패턴 장애물이 누적 추가된다(레벨 2부터 20까지 단계별 라인 패턴).
- 추가로 레벨마다 난수 장애물 `level * 2`개를 시드 고정 RNG로 배치한다.
- 시작 안전 구역(`x: 0..6`, `y: 8..12`)과 시작 스네이크 위치는 장애물 배치에서 제외한다.

## 참고
- 스테이지 클리어 시 다음 스테이지로 자동 이동, 마지막(20) 클리어 시 게임 완료.
- `R` 키 또는 재시작 버튼은 현재 스테이지를 재시작한다(총점은 유지).
